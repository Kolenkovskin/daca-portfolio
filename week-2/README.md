# Неделя 2 — Очистка данных (SQL Puhastamine)

**Задача (Toomas Kask, IT Director UrbanStyle):** найти и убрать дубли в таблице продаж перед заседанием правления. Kristi Tamm (CEO) требует чистые цифры, которым можно доверять.

## Процесс: Test → Verify → Log → Commit

| Шаг | Действие | Результат |
|---|---|---|
| Test | Копия `sales_test`, продакшн `sales` не тронут | 15234 строки |
| Диагностика | Дубли через `GROUP BY` + `HAVING count(*) > 1` | 4013 задвоенных `sale_id` |
| Масштаб | `count(*) − count(DISTINCT sale_id)` | 5116 лишних строк (33.58% таблицы) |
| Чистка | Дедуп через `ROW_NUMBER` + подзапрос, оставлен `rn = 1` | оригиналы |
| Commit | Таблица `sales_clean` | 15234 → 10118 |

## Находка

Дубли рождены **двойным импортом**: касса (POS) и сайт (e-commerce) грузили одни и те же данные повторно. `sale_date` хранится как **text** в разных форматах (`2023-01-16` ISO vs `16/01/2023` европейский), поэтому база не распознала строки как один день — отсюда задвоения. Один дефект (двойной импорт) вскрылся через другой (рассинхрон форматов даты).
